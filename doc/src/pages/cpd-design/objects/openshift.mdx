# OpenShift configuration

## `roks` - Red Hat OpenShift Kubernetes Service on IBM Cloud
<InlineNotification kind="warning">
The roks entry will be replaced by "openshift" soon to allow for other infrastructure types.
</InlineNotification>

```
roks:
- name: sample
  kube_version: 4.6_openshift
  worker_flavour: bx2.16x64
  worker_count: 1
  resource_group_name: ibm
  max_worker_count: 10
  infrastructure:
    type: vpc
    vpc_name: sample
    subnets: 
    - sample-subnet-zone-1
    - sample-subnet-zone-2
    - sample-subnet-zone-3
  storage_classes:
  - storage_type: nfs
    nfs_server_name: sample-nfs
  - storage_type: ocs
    ocs_storage_label: ocs
    ocs_storage_size: 500Gi
```

### Property explanation
| Property | Description                                                          | Mandatory | Allowed values |
| -------- | -------------------------------------------------------------------- | --------- | -------------- |
| name     | Name of the OpenShift cluster                                        | Yes       |  |
| kube_version | ROKS Kubernetes version                                          | Yes       | 4.6-openshift  |
| worker_flavour | Type of worker to be used                                      | Yes       | [link]  |
| worker_count | Number of workers per subnet. Eventual number of workers will be value multiplied by number of subnets | Yes | Integer |
| resource_group_name | IBM Cloud resource group for the ROKS cluster             | Yes, inferred  |  |
| max_worker_count | Maximum number of workers allowed. If total number of workers exceeds this value, the cluster will not be provisioned | Yes | Integer |
| infrastructure.type | Type of infrastructure to provision ROKS cluster on | No | vpc (default), classic |
| infrastructure.vpc_name | Name of the VPC if type is `vpc` | Yes, inferrred | Existing VPC |
| infrastructure.subnets | List of subnets within the VPC to use. Either 1 or 3 subnets must be specified | Yes | Existing subnet |
| storage_classes.storage_type | Type of storage class to create in the OpenShift cluster | Yes | nfs or ocs |
| storage_classes.nfs_server_name | Name of the NFS server within the VPC | Yes if `storage_type` is `nfs` | Existing `nfs_server` |
| storage_classes.ocs_storage_label | Label to be used for the dedicated OCS nodes in the cluster | Yes if `storage_type` is `ocs` |  |
| storage_classes.ocs_storage_size | Size of the OCS storage in `Gi` | Yes if `storage_type` is `ocs` |  |
